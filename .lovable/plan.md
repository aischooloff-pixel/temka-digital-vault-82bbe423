## КРИТИЧНО (до релиза обязательно)

### 1. /start работает, но `accepted_terms` не выставляется
Код `/start` (строка 1184-1227) корректен: отправляет приветствие, upsert профиля, загрузка аватара. Но в upsert нет `accepted_terms: true` — колонка добавлена миграцией, но никогда не устанавливается. Нужно добавить `accepted_terms: true` в объект upsert (строка 1218).

### 2. Ссылка поддержки захардкожена на `paveldurov`
- `Account.tsx` строка 300: `https://t.me/paveldurov`
- `OrderSuccess.tsx` строка 173: `https://t.me/paveldurov`

Бот корректно читает `support_username` из `shop_settings`, а фронтенд — нет. Нужно: создать хук `useSupportUsername`, который запрашивает `shop_settings` по ключу `support_username`, и подставлять результат в обе страницы.

### 3. Нет проверки наличия товара при оплате
Ни `create-invoice`, ни `pay-with-balance` не проверяют `stock > 0` перед созданием заказа. Пользователь может оплатить товар, которого нет на складе. Нужно добавить проверку в обе edge-функции.

### 4. Нет проверки `is_blocked` при оплате
Заблокированный пользователь может оформить заказ. Нужно проверять `is_blocked` в `create-invoice` и `pay-with-balance`.

### 5. Race condition при двойной обработке оплаты
`cryptobot-webhook` и `check-payment` содержат одинаковую логику автодоставки. Если webhook и polling сработают одновременно, инвентарь может задвоиться. Нужно: в `check-payment` перед обработкой проверять `payment_status === "paid"` (уже есть, строка 51-55, ОК), но в логике автодоставки нет блокировки от параллельного выполнения. Минимальный фикс: в обеих функциях перед назначением инвентаря проверять, нет ли уже `sold` единиц с `order_id = текущий заказ`.

## СРЕДНИЙ ПРИОРИТЕТ (можно отложить)

### 10. Дублирование логики автодоставки
Одинаковый ~60-строчный блок в `cryptobot-webhook`, `check-payment`, `pay-with-balance`. Любой багфикс нужно применять 3 раза. Рефакторинг в shared-модуль — после запуска.

### 11. Промокод не очищается после успешной оплаты
Промокод остаётся в localStorage. Server-side проверка `max_uses_per_user` защищает, но UX неидеален.

### 12. Нет минимальной суммы заказа
После промокода+баланса итог может быть $0.00 для CryptoBot, который может отклонить нулевой инвойс. (Для полной оплаты балансом — ОК.)

### 13. forwardRef warnings в консоли
`OrderDetailSheet`, `BalanceDetailSheet` — предупреждения React. Косметика, не критично.
